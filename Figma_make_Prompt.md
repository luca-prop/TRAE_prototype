1. 시스템 프롬프트 개선
현재 문제점:

shadcn/ui와 Figma Make 환경이 혼재되어 있음
Figma Make는 shadcn/ui를 기본 제공하지 않음
개선된 시스템 프롬프트:

너는 React, Tailwind CSS v4, TypeScript 기반의 고품질 UI를 작성하는 프론트엔드 수석 개발자야.
모든 컴포넌트는 반응형(Mobile-first)으로 작성하고, 아이콘은 `lucide-react`를 사용해.
클라이언트 상태가 필요한 인터랙티브 UI는 "use client"로 명시해.
재사용 가능한 컴포넌트는 src/app/components에 분리하고, 일관된 디자인 시스템을 유지해.
금액은 항상 천 단위 콤마 포맷팅을 적용하고, 날짜는 YYYY.MM.DD 형식을 사용해.
지금부터 내가 요청하는 화면 단위 요구사항에 맞춰 깔끔하고 모던한 UI 코드를 생성해줘.

개선된 프롬프트 1:

B2C 사용자를 위한 '가용 현금 기반 재개발 구역 검색' 페이지 뷰를 만들어줘.

1. 레이아웃 (Layout)
- 페이지 하단 고정 Footer: "⚠️ 본 데이터는 국토부 실거래가 및 동일 행정구 기준 비교이며, 현장 호가와 다를 수 있습니다. 투자 판단의 책임은 이용자에게 있습니다."

2. 랜딩 페이지 영역 (Hero Section)
- 화면 정중앙 헤딩: "내 가용 현금으로 진입 가능한 재개발 구역은?"
- 금액 입력창(Input): 
  * 실시간 천 단위 콤마 포맷팅 (예: 50000000 → 50,000,000)
  * placeholder: "예산을 입력하세요 (예: 50,000,000원)"
  * 최소값 1,000,000원, 최대값 10,000,000,000원 유효성 검사
  * 숫자 외 입력 차단
- '검색' 버튼: Primary 버튼, 금액 미입력 시 disabled 상태

3. 검색 결과 리스트 (검색 후 렌더링)
- 로딩 상태: 검색 중 Skeleton Card 3개 표시
- 데이터 신선도 경고: `last_synced_at`이 30일 초과 시 상단에 Warning Alert
  * "⚠️ 마지막 데이터 갱신일: {YYYY.MM.DD} ({N}일 전). 데이터가 최신이 아닐 수 있습니다."
- 결과 카드 구성:
  * 구역명 (h3)
  * 행정구 (회색 Badge)
  * 사업 단계 (Color Badge: 착공준비-파랑, 조합설립-초록, 관리처분-주황)
  * 예상 실투자금 범위: "3,500만원 ~ 5,200만원" (천 단위 콤마)
  * 매칭 적합도: Progress Bar (0~100%, 80% 이상-초록, 50~80%-노랑, 50% 미만-회색)
  * 카드 클릭 시 상세 페이지 이동 (href 준비)
- Pagination: 페이지당 10개, 총 페이지 수 표시

4. Empty State (매칭 0건)
- 중앙 정렬 Empty UI
- 헤딩: "현재 예산 범위 내 진입 가능한 구역이 없습니다."
- 서브텍스트: "예산을 조정하거나 알림을 설정해보세요."
- Primary CTA: "예산 내 매물 출현 시 알림 받기" (클릭 시 이메일 입력 Dialog)
- Secondary 링크: "예산 수정하기" (클릭 시 입력창으로 smooth scroll)

5. 에러 상태 처리
- 검색 API 실패 시: "일시적인 오류가 발생했습니다. 다시 시도해주세요." Destructive Alert + 재시도 버튼

개선된 프롬프트 2:

재개발 구역과 기축 아파트를 1:1 비교하는 '대시보드 및 심층 리포트' 페이지를 만들어줘.

1. 반응형 레이아웃
- 데스크톱(≥768px): 2단 컬럼 (좌측 재개발 정보 40%, 우측 기축 비교 60%)
- 모바일(<768px): 상하 스택, 재개발 카드 → 기축 리스트 순서

2. 상단 알림 영역
- 데이터 범위 확장 알림 (조건부 렌더링):
  * "ℹ️ {원래 행정구} 내 비교 데이터가 부족하여, 인접 행정구({확장된 구역명}) 기준으로 산정되었습니다."
  * Info Alert (파란색 배경)
- 예산 부족 경고 (조건부 렌더링):
  * "⚠️ 현재 가용 현금이 최소 진입 비용보다 {부족 금액}원 부족합니다."
  * Warning Alert (노란색 배경)
  * 하단에 예산 조정 Slider 표시 (min: 현재 값, max: 권장 값, step: 100만원)
  * Slider 값 변경 시 실시간으로 대시보드 데이터 업데이트 (Debounce 500ms)

3. 좌측: 재개발 구역 정보 카드
- 구역명 (h2, font-bold)
- 주요 지표 (Grid 2열):
  * 비례율: "120%" (Badge)
  * 권리가액: "5억 2,000만원"
  * 예상 추가 부담금: "3,500만원"
  * 입주 예정: "2028년 3분기"
- 잠재 미래 가치 섹션:
  * "참조 단지: {신축 단지명}"
  * "예상 시세: {금액 범위}" (회색 텍스트)
  * "상승 여력: {전고점 대비 %}%" (초록색 강조)

4. 우측: 기축 아파트 비교군 (3개 리스트)
각 아파트 카드 구성:
- 아파트명 + 동/평형 (h3)
- 최신 실거래가: "6억 5,000만원" (2026.03 기준)
- 전고점 데이터: "7억 2,000만원 (2021.08)"
- 회복률 Progress Bar + Badge:
  * ≥80%: 초록색 "회복 우수"
  * 60~80%: 노란색 "회복 중"
  * <60%: 빨간색 "미회복"
- 우측 상단: "변경" 버튼 (작은 Ghost 버튼)

5. 아파트 변경 Dialog
- 트리거: "변경" 버튼 클릭
- Dialog 내부:
  * 상단 Input: "아파트명 또는 단지명 검색" (실시간 필터링)
  * 검색 결과 리스트 (최대 10개, Scroll)
  * 각 항목: 아파트명, 최근 거래가, Radio 선택
  * 하단: "취소" / "적용" 버튼
- "적용" 클릭 시: Dialog 닫기 + 해당 카드만 업데이트 (부드러운 전환 애니메이션)

6. 심층 분석 리포트 (토글 영역)
- 트리거: 대시보드 하단 "📊 심층 분석 리포트 보기" 버튼
- 열림 상태:
  * 4개 섹션을 Accordion 또는 Tab으로 구성
  * 섹션: "투자 구조 비교", "주거 비용 비교", "현금 흐름 시뮬레이션", "미래 가치 분석"
  * 각 섹션은 Table 컴포넌트로 렌더링 (2~3열, 5~8행)
  * 로딩 중: Table Skeleton (3행 shimmer 효과)
  * 각 테이블 우측 상단: "CSV 다운로드" 아이콘 버튼

7. 인터랙션 피드백
- 모든 버튼 클릭 시 ripple 효과
- 카드 호버 시 미세한 그림자 강화 (transition 200ms)
- Slider 조작 시 값 표시 Tooltip

1. 매물 등록 폼 ("use client" 컴포넌트)
기술 스택:
- Form 라이브러리: react-hook-form v7.55.0
- Validation: zod
- 상태 관리: useState (로컬 상태)

필드 구성 (세로 1열 레이아웃):
a) 구역 선택 (Select, required)
   - placeholder: "재개발 구역 선택"
   - 옵션: 서버에서 받은 구역 리스트
   - 에러: "구역을 선택해주세요"

b) 매물 유형 (Select, required)
   - 옵션: "뚜껑", "다세대", "빌라", "상가", "기타"
   - 에러: "매물 유형을 선택해주세요"

c) 호가 (Input, required, number)
   - 천 단위 콤마 포맷팅 (onChange 시 실시간 적용)
   - placeholder: "예: 350,000,000"
   - 최소값: 1,000,000원
   - 서버 에러 처리: 이상치 감지 시 필드 하단에 빨간색 텍스트
     * "⚠️ 정상 범위를 벗어난 호가입니다. 오타를 확인해 주세요."
     * Input border를 빨간색으로 변경

d) 프리미엄 (Input, optional, number)
   - 천 단위 콤마 포맷팅
   - placeholder: "권리금이 있다면 입력 (선택)"

e) 권리가액 (Input, required, number)
   - 천 단위 콤마 포맷팅
   - 에러: "권리가액을 입력해주세요"

f) 소유주 연락처 (Input, required, tel)
   - 자동 하이픈 삽입 (010-1234-5678)
   - pattern validation: 010으로 시작하는 11자리
   - 에러: "올바른 휴대폰 번호를 입력해주세요"

g) 동호수 (Input, optional)
   - placeholder: "예: 101-1001"

제출 버튼:
- Primary 버튼 "매물 등록하기"
- 제출 중: 버튼 disabled + 로딩 스피너 + "등록 중..." 텍스트
- 성공 시: Toast 알림 "매물이 등록되었습니다" + 폼 초기화
- 실패 시: Toast 알림 + 에러 메시지 표시

접근 차단 상태 (패스코드 불일치):
- 폼 최상단에 Destructive Alert:
  * "🚫 유효하지 않은 접근 코드입니다. 관리자에게 문의하세요."
- 모든 Input과 Select: disabled={true}
- 제출 버튼: disabled={true}, 회색 처리
- 배경 흐림 효과 (opacity-60)

3. LTV 정책 관리 폼 (동적 폼)
"use client" 컴포넌트, react-hook-form 사용

테이블 형태의 폼:
- 헤더: Tier | 가격 하한 | 가격 상한 | 최대 대출액(%) | 적용일 | 삭제
- 각 행(Tier):
  * Tier 라벨: Input (예: "일반", "우대", "특별")
  * 가격 하한/상한: Input (천 단위 콤마)
  * 최대 대출액: Input (%) + Slider (0~100%)
  * 적용일: Date Input
  * 삭제: 빨간색 Trash 아이콘 버튼

동적 필드 관리:
- "Tier 추가" 버튼 (테이블 하단): 클릭 시 새 행 추가 (최대 10개)
- 삭제 버튼: 클릭 시 해당 행 제거 (최소 1개 유지)
- Validation: 가격 범위 겹침 검사 (하한 < 상한, Tier 간 중복 없음)

저장 버튼:
- Primary 버튼 "정책 저장"
- 저장 중: 로딩 상태 + "저장 중..."
- 성공: Toast "LTV 정책이 업데이트되었습니다"
- 실패: Destructive Alert + 에러 메시지

4. 매물 관리 테이블 (Data Table)
컬럼 구성:
- 구역 (정렬 가능)

- 호가 (정렬 가능, 천 단위 콤마)
- 프리미엄 (천 단위 콤마)
- 상태 (ACTIVE/SOLD, 인라인 편집 Select)
- 연락처 (마스킹 없음, 클릭 시 복사 기능)

- Actions (Dropdown Menu)

Actions Dropdown 항목:

6. 프롬프트 5 개선사항
개선된 프롬프트 5:

B2C 사용자가 특정 구역의 등록된 매물들을 보는 '매물 리스트 뷰'를 작성해줘.

1. 페이지 구조
- 상단: 구역명 + 총 매물 수 표시 (h1)
  * "도봉구 쌍문동 재개발 구역 - 총 12개 매물"
- 정렬 옵션 (Select): "최신순", "가격 낮은순", "가격 높은순"

2. Verified 매물 우선 정렬
매물 카드는 두 종류로 구분:

A) 인증된 매물 카드 (리스트 최상단 고정)
- 테두리: border-2 border-green-500
- 좌측 상단: "✅ Verified" Badge (초록색 배경, 흰색 텍스트)
- 배경: 미세한 초록색 tint (bg-green-50)
- 우측 상단: "중개사 인증" 작은 텍스트 (회색)

B) 일반 매물 카드
- 기본 Card 스타일 (회색 테두리)
- Badge나 배경색 없음

3. 카드 정보 렌더링 (공통)
각 카드에 표시할 내용:
- 매물 유형 (Badge): "뚜껑", "다세대" 등
- 호가: "3억 5,000만원" (큰 bold 텍스트)
- 프리미엄: "+2,000만원" (있을 경우만, 작은 텍스트)
- 권리가액: "5억 2,000만원" (회색 텍스트)
- 등록일: "2026.04.15" (우측 하단, 작은 회색 텍스트)
- "상세 보기" 버튼 (Ghost 버튼, 카드 하단 우측)

보안 처리 (중요):
- ❌ 소유주 연락처: UI 코드에서 완전히 제외
- ❌ 동호수: UI 코드에서 완전히 제외
- ❌ 상세 주소: UI 코드에서 완전히 제외
- "상세 보기" 버튼은 별도 인증 후 연락처 제공 플로우로 연결 (Dialog)

4. 연락처 요청 Dialog
- 트리거: "상세 보기" 버튼 클릭
- Dialog 내용:
  * "이 매물의 상세 정보를 받으시겠습니까?"
  * "연락처 정보는 이메일로 전송됩니다."
  * Input: 이메일 주소 입력
  * "정보 요청하기" Primary 버튼
- 제출 시: Toast "요청이 접수되었습니다. 이메일을 확인해주세요."

5. 리스트 레이아웃
- 데스크톱: 2열 Grid (gap-6)
- 모바일: 1열 Stack
- 카드 호버: 미세한 그림자 + 약간 위로 이동 (transform translateY(-2px))

6. 하단 Pagination
- 페이지당 10개 매물
- 현재 페이지 하이라이트
- "이전" / "다음" 버튼

7. Empty State (매물 0개)
- 중앙 정렬 메시지: "이 구역에는 아직 등록된 매물이 없습니다."
- "관심 구역 알림 신청" 버튼 (이메일 입력 Dialog)
