⚙️ [공통 설정] 시스템 프롬프트 (System Prompt)
UI 생성 AI 도구의 시스템 설정이나 첫 대화에 입력하여 기본 컨텍스트를 잡아주세요.

Plaintext
너는 Next.js (App Router), Tailwind CSS, TypeScript, 그리고 shadcn/ui 기반의 고품질 UI를 작성하는 프론트엔드 수석 개발자야. 
모든 컴포넌트는 반응형(Mobile-first)으로 작성하고, 아이콘은 `lucide-react`를 사용해. 
서버 컴포넌트(RSC)와 클라이언트 컴포넌트("use client")를 명확히 구분해서 코드를 작성하고, 상태 관리가 필요한 폼이나 인터랙티브 UI는 클라이언트 컴포넌트로 분리해. 
지금부터 내가 요청하는 화면 단위 요구사항에 맞춰 깔끔하고 모던한 UI 코드를 생성해줘.


📱 프롬프트 1: B2C 메인 랜딩 & 검색 결과 리스트 (FE-F-001 ~ FE-F-005)
Plaintext
B2C 사용자를 위한 '가용 현금 기반 재개발 구역 검색' 페이지 뷰를 만들어줘. 아래 요구사항을 모두 포함해야 해.

1. 레이아웃 (Layout)
- 페이지 하단에 항상 고정되는 Footer를 만들어줘. 텍스트: "⚠️ 본 데이터는 국토부 실거래가 및 동일 행정구 기준 비교이며, 현장 호가와 다를 수 있습니다. 투자 판단의 책임은 이용자에게 있습니다."

2. 랜딩 페이지 영역 (Hero Section)
- 복잡한 지도 없이 화면 정중앙에 "내 가용 현금으로 진입 가능한 재개발 구역은?" 이라는 큰 헤딩을 배치해.
- 중앙에 단일 금액 입력창(Input)과 '검색' 버튼을 배치해. 금액을 입력하면 실시간으로 1,000 단위 콤마가 찍히도록 포맷팅해줘.

3. 검색 결과 리스트 영역 (검색 후 렌더링)
- 상단 배너: 만약 `last_synced_at` 데이터가 30일을 초과했다면 "⚠️ 마지막 데이터 갱신일: 2026.03.01 (45일 전). 데이터가 최신이 아닐 수 있습니다." 라는 Warning 형태의 Alert 컴포넌트를 상단에 띄워줘.
- 결과 카드 리스트: 검색된 구역들을 Card 컴포넌트로 나열해. 각 카드에는 [구역명, 행정구(Badge), 사업 단계(Color Badge), 예상 실투자금 범위(원화), 매칭 적합도(Progress Bar)]가 들어가야 해.
- 페이지네이션: 카드 목록 하단에 shadcn/ui 기반 Pagination을 추가해.

4. Empty State (매칭 0건일 때)
- 검색 결과가 없을 때 빈 카드 리스트 대신 중앙에 빈 상태 UI를 보여줘.
- 텍스트: "현재 예산 범위 내 진입 가능한 구역이 없습니다."
- CTA 버튼: "예산 내 매물 출현 시 알림 받기" (Lead Gen 목적의 Primary 버튼).
- 서브 링크: "예산 수정하기" (클릭 시 입력창으로 포커스).


📊 프롬프트 2: B2C 1:1 대조 대시보드 및 리포트 (FE-G-001 ~ FE-G-005)
Plaintext
재개발 구역과 기존 아파트를 1:1로 비교하는 '대시보드 및 심층 리포트' 화면을 만들어줘. 데스크톱에서는 2단 컬럼(좌/우), 모바일에서는 상하 스택으로 반응형 처리해.

1. 대시보드 상단 경고/안내 툴팁
- 만약 동일 행정구에 데이터가 없어서 인접 구역으로 확장되었다면, 상단에 Info Alert으로 "ℹ️ 도봉구 내 비교 데이터가 부족하여, 인접 행정구(노원구) 기준으로 산정되었습니다."를 띄워줘.
- 사용자의 가용 현금이 부족할 경우(LTV 미달), Warning Alert을 띄우고, 그 아래에 예산을 조정할 수 있는 shadcn/ui `Slider`를 배치해. (최소~최대값 표시).

2. 좌/우 비교 카드 레이아웃
- 좌측 (재개발 구역 정보): 구역명, 비례율, 권리가액, "잠재 미래 가치 (참조 단지명)"을 표시하는 큰 Card.
- 우측 (기축 아파트 비교군): 추천된 기축 아파트 3개를 세로 리스트 형태로 렌더링.
  - 각 아파트 카드에는 최신 실거래가, 전고점 비교 데이터가 들어감.
  - 전고점 대비 회복률 배지를 추가해 (80% 이상: 초록색, 60~80%: 노란색, 60% 미만: 빨간색).
  - 각 아파트 카드 우측 상단에 "변경" 버튼을 추가해.

3. 커스텀 편집 Dialog (모달)
- 아파트 카드의 "변경" 버튼을 누르면 shadcn/ui Dialog가 열림.
- 내부에 아파트를 검색할 수 있는 Input과 검색 결과 리스트가 있고, 선택 후 '적용' 버튼을 누르는 UI를 구성해.

4. 심층 분석 리포트 영역
- 대시보드 하단의 "심층 분석 리포트 보기"를 누르면 나오는 영역이야.
- 4개의 섹션(투자 구조, 주거 비용, 현금 흐름, 미래 가치)을 테이블(Table) 형태로 렌더링해.
- 각 테이블은 React Suspense로 감싸진 상태를 가정하고, 데이터 로딩 중일 때 보여줄 세련된 Skeleton UI도 함께 작성해줘.


🛡️ 프롬프트 3: B2B 매물 등록 폼 & 고객 브리핑 모드 (FE-H-001 ~ FE-H-003, FE-H-005)
Plaintext
B2B 중개사를 위한 '매물 등록 폼'과 '고객 브리핑 모드' UI를 작성해줘.

1. B2B 매물 등록 폼 ("use client", react-hook-form + zod 사용)
- 필드 구성: 구역 선택(Select), 매물 유형(Select - 뚜껑/다세대/빌라/기타), 호가(Input), 프리미엄(Input), 권리가액(Input), 소유주 연락처(Input), 동호수(Input). 금액 필드는 1,000 단위 콤마 포맷팅 적용.
- 이상치 에러 상태: 호가 필드 하단에 서버에서 내려온 에러 메시지("정상 범위를 벗어난 호가입니다. 오타를 확인해 주세요.")를 빨간색 텍스트로 보여줄 수 있는 상태를 구현해.
- 패스코드 불일치 (접근 차단) 상태: 폼 상단에 "유효하지 않은 접근 코드입니다" Destructive Alert을 띄우고, 폼 내부의 모든 Input과 Button이 `disabled` 처리된 상태를 코드로 보여줘.

2. B2B 고객 브리핑 뷰 (Read-only)
- 중개사가 고객에게 화면을 보여줄 때 사용하는 브리핑 모드 뷰야.
- 상단에 "🎤 고객 브리핑 모드" 배지를 달아줘.
- 매물 상세 정보가 보이는데, 보안상 소유주 연락처는 '010-****-5678', 동호수는 '***-****' 형태로 마스킹(Masking) 처리해서 텍스트를 렌더링해.
- 하단에는 투자 시뮬레이션(마스킹 없는 온전한 차트/테이블 데이터)이 보인다고 가정하고 레이아웃을 짜줘.


👑 프롬프트 4: Admin 관리자 대시보드 및 데이터 테이블 (FE-I-001 ~ FE-I-004)
Plaintext
Admin 관리자를 위한 시스템 관리 대시보드 UI를 만들어줘. 사이드바 네비게이션(대시보드, LTV 관리, 매물 관리, 구독 관리)이 있는 레이아웃을 전제해.

1. 메인 대시보드 요약 (Hero)
- 상단에 5개의 시스템 현황 요약 카드(Stat Card)를 그리드(Grid)로 배치해.
- 항목: 총 구역 수, 총 매물 수, Verified 비율(%), 최근 동기화 시간, Lead 구독 수.

2. LTV 정책 관리 폼 (3-Tier CRUD)
- 3개의 Tier(행)를 관리하는 테이블 형태의 폼.
- 각 행의 컬럼: Tier 라벨, 가격 하한, 가격 상한, 최대 대출액, 적용일, [삭제 버튼].
- 하단에 "Tier 추가" 버튼과 폼을 전송하는 "저장" 버튼을 배치해. (동적 폼 필드 추가/삭제 UI)

3. 매물 관리 테이블
- shadcn/ui의 Data Table 컴포넌트를 사용해.
- 컬럼: 구역, 매물유형, 호가, 프리미엄, 상태(Select로 ACTIVE/SOLD 변경 가능), 연락처(마스킹 없음), 등록일, Actions.
- Actions 컬럼 안에는 Dropdown Menu를 넣고, "✅ Verified 토글", "🗑️ 매물 삭제" 버튼을 넣어줘.

4. Lead 구독 관리 테이블
- 컬럼: 예산, 타겟 지역, 상태, 등록일.
- 테이블 상단에 '예산 범위(min~max)'를 필터링할 수 있는 Input 2개와 'CSV 내보내기' Button을 배치해줘.


💡 B2C 매물 리스트 컴포넌트 프롬프트 (FE-H-004)
Plaintext
B2C 사용자가 특정 구역의 등록된 매물들을 보는 '매물 리스트 뷰'를 작성해줘.

1. Verified 매물 우선 정렬 UI
- 매물 카드 리스트를 렌더링하는데, 카드는 두 종류로 나뉘어.
- '인증된 매물 카드': 리스트 최상단에 고정. 초록색 테두리(border-green-500)가 있고, 상단에 "✅ Verified" 배지가 돋보이게 붙어 있어야 해.
- '일반 매물 카드': 인증 매물 아래에 나열됨. 뱃지나 강조 테두리 없음.

2. 카드 정보 렌더링
- 카드 내부에는 매물 유형, 호가, 프리미엄, 권리가액 데이터만 표시해.
- (중요 보안) 카드 내부 어디에도 소유주 연락처, 동호수, 상세 주소가 노출되지 않도록, 해당 필드 자체를 UI 코드에서 배제해줘.
- 리스트 하단에 Pagination 컴포넌트를 배치해.
